# Projects

Concrete work with clear deliverables. Unlike goals (aspirations), projects have a definite finish line.

## Structure

| Location | Purpose |
|----------|---------|
| `05_Projects/` (root) | Active projects (limit to 2-3) |
| `Backlog/` | Planned but not started |
| `Archive/` | Completed or paused projects |

## Project Lifecycle

1. **Idea** → Create in `Backlog/`
2. **Start** → Move to root `05_Projects/`
3. **Ship** → Move to `Archive/`

## Project File Format

```markdown
# Project Name

**Goal:** [[Related Goal]] (optional)
**Status:** Active | Backlog | Archived
**Started:** YYYY-MM-DD
**Target:** YYYY-MM-DD (optional)

## What

[One sentence: what will exist when this is done?]

## Why

[Connection to goals, motivation]

## Deliverables

- [ ] Deliverable 1
- [ ] Deliverable 2
- [ ] Deliverable 3

## Progress Log

### YYYY-MM-DD
[Update]

## Notes

[Any relevant context, decisions, blockers]
```

## Goals vs Projects

| Goals | Projects |
|-------|----------|
| Aspirations | Work |
| Long-term direction | Specific deliverable |
| May never "complete" | Has a finish line |
| `03_Goals/` | `05_Projects/` |

## Examples

| Goal | Project |
|------|---------|
| Become a published author | Write first novel |
| Build passive income | Launch online course |
| Get fit | Complete 12-week program
